## [2026-07-17] add | Part 11 — Local and Open-Source Models on the Endpoint (Cline, LM Studio, Ollama)

Added `wiki/part-11-local-open-source-models.md`. Core thesis: running open-weights models locally feels private and is therefore treated as safe, and is neither. Three risk clusters: (1) the governance bypass — local inference on `localhost` means the AI gateway (Parts 1/7) sees nothing, so prompt/response logging, PII/secret screening, the model allowlist, spend limits, and the IR audit trail are all blind, and the false sense of privacy invites more sensitive inputs; (2) the model-file supply chain — pickle-format weights execute code on load (100+ malicious models on a major hub, still appearing through 2025), GGUF has carried Jinja-template payloads in metadata, model scanners are bypassable (e.g. CVE-2025-10155), and the runtime itself has had RCEs (Ollama Probllama CVE-2024-37032, ZipSlip CVE-2024-7773); (3) the autonomous agent with safety off — Cline (and Roo/Kilo/OpenHands) runs shell/files/browser, and YOLO mode disables every check, plus the local inference server (LM Studio) is unauthenticated, can bind 0.0.0.0, and has open CORS. Playbook: reframe private≠safe; prefer org-hosted open weights behind the gateway; if truly local, sanction and log the runtime; govern model files as executable artifacts (Part 6); lock the server (127.0.0.1, auth, patched); keep the agent controls (Part 2 allowlist/managed-settings/hook/sandbox); inventory local runtimes as shadow AI (Part 1); instrument endpoint detection for the lost trail (Part 9); mind model licenses.

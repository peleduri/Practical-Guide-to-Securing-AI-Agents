## [2026-07-17] add | Part 8 — Enterprise Work AI and the DSPM Prerequisite (Glean and peers)

Added `wiki/part-8-work-ai-and-dspm.md` on enterprise Work AI platforms (Glean, Copilot/Gemini Enterprise assistant side). Core thesis: these platforms are permission-aware, which is exactly the risk — they faithfully mirror overshared permissions (anyone-with-link, stale ACLs, bloated all-company groups), turning years of quiet oversharing into a natural-language query, so an engineer can surface HR comp / M&A / marketing content they were technically entitled to but never should see. DSPM is the non-optional companion: discover and classify sensitive data, map who-can-access-what, remediate oversharing at the source before connecting the corpus, deliberately map audience-to-content, and run continuously because exposure drifts. References Glean Protect and Microsoft Purview DSPM for AI. Updated `index.md` and `README.md`.

## [2026-07-17] add | Part 11 — Local and Open-Source Models on the Endpoint (Cline, LM Studio, Ollama)

Added `wiki/part-11-local-open-source-models.md`. Core thesis: running open-weights models locally feels private and is therefore treated as safe, and is neither. Three risk clusters: (1) the governance bypass — local inference on `localhost` means the AI gateway (Parts 1/7) sees nothing, so prompt/response logging, PII/secret screening, the model allowlist, spend limits, and the IR audit trail are all blind, and the false sense of privacy invites more sensitive inputs; (2) the model-file supply chain — pickle-format weights execute code on load (100+ malicious models on a major hub, still appearing through 2025), GGUF has carried Jinja-template payloads in metadata, model scanners are bypassable (e.g. CVE-2025-10155), and the runtime itself has had RCEs (Ollama Probllama CVE-2024-37032, ZipSlip CVE-2024-7773); (3) the autonomous agent with safety off — Cline (and Roo/Kilo/OpenHands) runs shell/files/browser, and YOLO mode disables every check, plus the local inference server (LM Studio) is unauthenticated, can bind 0.0.0.0, and has open CORS. Playbook: reframe private≠safe; prefer org-hosted open weights behind the gateway; if truly local, sanction and log the runtime; govern model files as executable artifacts (Part 6); lock the server (127.0.0.1, auth, patched); keep the agent controls (Part 2 allowlist/managed-settings/hook/sandbox); inventory local runtimes as shadow AI (Part 1); instrument endpoint detection for the lost trail (Part 9); mind model licenses. Cross-links Parts 1/2/3/4/6/7/9/10. Renumbered the README Roadmap's Governance & Compliance item off the now-taken Part 11 slot. Updated `index.md`, `README.md`, `glossary.md`, `llms.txt`, and Part 10's Nav footer.
